Week 4: CSS basics

Published on October 6, 2019

I learned to:

  • Copy stylesheet.css to the webpages
  • Style the body element
  • Style the other elements
  • Validate the css stylesheet

A basic stylesheet template normalize.css was provided for us to copy into our own css file. I placed the link to the stylesheet (link href="css/normalize.css" rel="stylesheet") within the head tags along with the stylesheet link (link href="css/stylesheet" rel="stylesheet").
Selectors were also added to match the element tags located in the HTML doc. Later in the tutorial, I styled the navigation menu containing the Home, Tutorial and Contact by giving the nav element a class "menu". This would make it easier to target this element in the CSS.

So the nav in the HTML doc would be (nav class="menu") while the CSS target would become .menu li or .menu a depending on the style we're after.

Week 6: Making the Learning Journal page responsive

Published on October 26, 2019

I learned to:

  • Added a viewport meta element to the head of the document
  • Organised the stylesheet
  • A responsive grid was setup
  • Styled a small screen layout
  • Added media queries
  • Hiding elements display:none

In order to make the webpage i just created act like a normal desktop page and dusplay the contents in full width, I added this line of code into the head: (meta name="viewport" content="width=device-width, initial-scale=1"). Following this, adjusted the order of HTML elements and reorganised the sections of the stylesheet for easier layout. Firstly I added a base style layout which would be applied to all the screens and webpages. Next is the Grid layout which was applied to only the Learning Journal Page. Now this layout contained a grid of 3 columns below the banner, nav, table, main, aside and footer. These areas just mentioned are the content areas but I hadn't created the layout yet. If this webpage is opened on a mobile device, the nabvigation menu would resemble a vertical menu, and the reason for this is because in the .menu target in the css, it contains display: block; to display it in a vertical column. While for the banner, display: inline; styles a horizontal menu for bigger and wider screens.

Week 9: HTML & CSS Forms

Published on November 25, 2019

I learned to:

  • Insert a form on a webpage
  • Add form elements to the contact form; text input elements, text-area, fieldset and radio group
  • Enhancing forms with HTML5 input types and other features
  • Validate HTML5 forms

I applied a fieldset tag in order to make a container for the area required for the user to input their contact details. Enclosed in these tags was the legend tag which represents the caption for this fieldset container. Following these tags was the label tags embedded within the fieldset to label the textboxes requiring the user to enter their details.

Below the section of the form where the user details are entered, there is also a textarea tag added for the text area. This is where the user enters additional comments. The textarea also has an id attribute message.
